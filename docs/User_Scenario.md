# 마이링크 (MyLink) 사용자 시나리오 (User Scenario)

본 문서는 마이링크 서비스의 핵심 페르소나인 '링크 방문자'와 '프로필 소유자(인플루언서)'의 주요 행동 패턴을 정의합니다. 모든 행동은 기획 의도와 목표를 명확히 하기 위해 **"사용자는 [목적]하기 위해 [행동]한다."** 형식으로 작성되었습니다.

---

## 1. 링크 방문자 (Visitor) 시나리오

### 1-1. 서비스 접근 및 정상 탐색
1. 방문자는 **인플루언서가 모아둔 여러 소셜 링크와 콘텐츠를 한 번에 확인하기 위해**, 인스타그램 프로필 등에 기재된 `mylink.com/username` 단일 링크를 **클릭(접속)한다**.
2. 방문자는 **인플루언서가 본인이 찾는 사람이 맞는지 파악하기 위해**, 프로필 상단에 표출된 닉네임과 짧은 소개글(Bio)을 **읽는다**.
3. 방문자는 **자신이 관심 있는 웹사이트나 목적지(예: 유튜브 영상, 블로그)로 이동하기 위해**, 나열된 여러 개의 링크 버튼 중 원하는 항목을 **터치(클릭)하여 아웃링크로 넘어간다**.
4. 방문자는 **해당 링크가 엉뚱한 사이트가 아닌지(신뢰도) 직관적으로 식별하기 위해**, 버튼 텍스트 좌측에 표출된 파비콘(아이콘) 이미지를 **확인한다**.
5. 방문자는 **마이링크 서비스의 운영 정책이나 정보 수집 약관을 열람하기 위해**, 화면 맨 밑바닥(Footer) 영역에 작게 쓰인 '이용약관' 또는 '개인정보 처리방침' 링크를 **누른다**.

### 1-2. 예외 상황 (Error Case)
1. 방문자는 **이미 삭제되었거나 아이디가 변경된 마이링크 주소로 잘못 접속했을 때 혼란을 겪지 않기 위해**, "존재하지 않는 프로필입니다"라는 안내 문구와 함께 메인 홈으로 돌아가는 버튼이 있는 **에러 페이지(404)를 확인한다**.

## 2. 프로필 소유자 (인플루언서) 시나리오

### 2-0. 초기 진입 (Onboarding)
1. 소유자는 **비밀번호 입력 없이 간편하고 빠르게 본인 계정으로 마이링크 서비스에 가입하기 위해**, 서비스 메인 화면 중앙의 구글 소셜 로그인 버튼을 **클릭한다**.
2. 소유자는 **자신만의 고유한 마이링크 접속 주소를 갖기 위해**, 로그인 직후 나타나는 온보딩 화면에서 원하는 영문/숫자 조합의 아이디(username)를 입력하고 **중복 확인을 누른다**.
3. 소유자는 **자신의 브랜드나 취향에 꼭 맞는 프로필 배경을 꾸미기 위해**, 이어지는 테마 설정 화면에서 원하는 색상 코드와 링크 버튼 모양 스타일을 **선택한다**.

### 2-1. 링크 관리 화면과 생성 (Create)
1. 소유자는 **팬들에게 공유할 새로운 타겟 웹페이지 주소를 등록하기 위해**, 대시보드 화면에 있는 '+ 링크 추가' 폼에 목적지 URL 주소를 복사 후 **붙여넣기 한다**.
2. 소유자는 **방문자가 해당 링크가 어떤 내용인지 쉽게 이해하도록 안내하기 위해**, 링크의 '제목(Title)' 입력 칸에 대표 문구를 직접 **타이핑한다**.
3. 소유자는 **입력·변경된 정보가 즉시 프로필 결과물 화면에 어떻게 보일지 예측하기 위해**, 화면 한편에 떠 있는 실시간 모바일 뷰어를 주기적으로 **주시한다**.

### 2-2. 트래픽 조회 및 통계 확인 (Read)
1. 소유자는 **자신이 개설한 마이링크 채널이 얼마나 성장하고 있는지 알아보기 위해**, 로그인 직후 대시보드 상단에 표기된 '전체 기간 뷰(PV)' 카운트를 **읽는다**.
2. 소유자는 **팬들이 특정 기간(최근 7일 등) 동안 활발하게 접속했는지 단기적인 트래픽 동향을 분석하기 위해**, 대시보드 내의 기간 필터 옵션을 마우스로 **조작한다**.
3. 소유자는 **자신이 등록한 여러 링크 중 팬들에게 가장 반응이 좋은 콘텐츠를 파악하기 위해**, 나열된 링크 블록별로 기록된 '클릭수' 수치를 **비교 확인한다**.

### 2-3. 링크 수정 및 정렬 (Update)
1. 소유자는 **가장 중요한 최신 동영상이나 이벤트를 프로필 최상단으로 끌어올려 집중적으로 노출하기 위해**, 해당 링크 블록 왼쪽의 점 6개 그립(Grip) 아이콘을 붙잡고 위쪽으로 **드래그 앤 드롭(Drag & Drop)하여 순서를 뒤바꾼다**.
2. 소유자는 **작성했던 제목에 오타가 생겼거나 목적지 URL 정보가 바뀌었을 때 이를 바로잡기 위해**, 기존 링크 블록의 텍스트 필드를 탭하고 문자를 **수정한다**.
3. 소유자는 **이벤트 기간이 끝난 프로모션 링크를 당분간 팬들의 접속만 임시로 차단하기 위해**, 링크 블록 우측 상단의 활성화 스위치를 클릭해 토글을 끄는(Off) 방식으로 링크를 **숨김 처리한다**.

### 2-4. 링크 삭제 (Delete)
1. 소유자는 **더 이상 쓸모가 없어진 링크를 데이터베이스에서 영구히 없애기 위해**, 대상 링크 블록의 휴지통 모양 아이콘을 **클릭한다**.
2. 소유자는 **마우스를 실수로 잘못 눌러 소중한 링크가 영구 삭제되는 것을 방지하기 위해**, 아이콘 클릭 직후 브라우저(또는 모달창)에 뜨는 "정말 삭제하시겠습니까?" 경고 창에서 '확인(또는 삭제)' 버튼을 재차 **누른다**.
